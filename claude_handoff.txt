## Key facts about the data

- **ACO**: stable at 10,000, std ~5. Mean-reverting. Only 2-4% of ticks have directly takeable mispricings. Core strategy is tight market-making with small inventory skew.
- **IPR**: near-perfect linear trend (fair = day_start + 0.001 × timestamp, R²=0.9999). Residual std 2.37, autocorrelation ≈ 0 (white noise around trend). Day starts rise by 1000 between days: ~11000/12000/13000 for days -1/0/1. Day 2 will likely start ~14000.
- Big volume market trades cluster at ±8 from mid for ACO, but most fills happen at ±2-4.

Instead of one 80-unit passive ask at `ceil(fair)+4`, split into two price levels and sweep the split point and the offsets. Requires code change to post two orders plus new regex entries for the split.

Recommend the user run this grid (integers only):

```python

    "ACO_SKEW": [2],
    "IPR_BUY_THRESH": [4],
    "IPR_SELL_THRESH": [6],
    "IPR_PASSIVE_BID": [1],
    "IPR_ASK_SPLIT": [30, 40, 50],        # units at tight tier (rest at wide)
```
